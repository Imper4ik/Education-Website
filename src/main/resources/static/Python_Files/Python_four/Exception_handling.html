Python Exception Handling

Introduction

Exception handling in Python allows you to handle runtime errors gracefully. It enables you to catch and manage exceptions that may occur during program execution.

Basic Syntax

The basic syntax for exception handling in Python uses `try`, `except`, `else`, and `finally` blocks:

def main():
    try:
        # Code that may raise an exception
        result = 10 / 0  # Division by zero
    except ZeroDivisionError as e:
        # Handling specific exception
        print("Error:", e)
    except Exception as e:
        # Handling general exception
        print("Unexpected error:", e)
    else:
        # Executed if no exception occurred
        print("Operation successful.")
    finally:
        # Always executed, cleanup or final operations
        print("End of program.")

if __name__ == "__main__":
    main()

This example demonstrates handling a `ZeroDivisionError` and a generic `Exception`, along with `else` and `finally` blocks.

Raising Exceptions

Raising exceptions explicitly:

def validate_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative.")
    elif age < 18:
        raise ValueError("Must be at least 18 years old.")

def main():
    try:
        validate_age(-5)
    except ValueError as e:
        print("Validation error:", e)

if __name__ == "__main__":
    main()

This snippet shows how to raise and handle custom exceptions using `raise`.

Exception Hierarchy

Python's exception hierarchy and handling multiple exceptions:

def main():
    try:
        # Code that may raise exceptions
        file = open("nonexistent.txt", "r")
        content = file.read()
    except FileNotFoundError as e:
        print("File not found error:", e)
    except (IOError, OSError) as e:
        print("I/O or OS error:", e)
    except Exception as e:
        print("Unexpected error:", e)

if __name__ == "__main__":
    main()

This example demonstrates handling specific exceptions like `FileNotFoundError` and grouping exceptions.

Summary

Exception handling in Python is crucial for managing errors and ensuring program stability. Using `try`, `except`, `else`, and `finally` blocks allows you to gracefully handle unexpected situations and clean up resources.
