Python Namespaces

Introduction

Namespaces in Python are containers for mapping names (identifiers) to objects. They help in avoiding name collisions and provide a way to organize variables, functions, and classes into separate scopes.

Types of Namespaces

Python has the following types of namespaces:

  • Local Namespace: Scope of a function or method. It includes local variables.
  • Global Namespace: Scope of the current module. It includes global variables and functions.
  • Built-in Namespace: Provided by Python itself. It includes built-in functions and exceptions.

Accessing Namespaces

You can access namespaces using:

  • locals() - Returns a dictionary representing the current local namespace.
  • globals() - Returns a dictionary representing the current global namespace.
  • dir() - Returns a list of names in the specified namespace.
# Example of accessing local and global namespaces
def example_func():
    local_var = 10
    print(locals())  # {'local_var': 10}

global_var = 20
print(globals())  # Includes global variables and functions

Namespace Resolution

Python follows a specific order (LEGB rule) to resolve namespaces:

  1. Local (L): Inside the current function or method.
  2. Enclosing (E): In any enclosing functions or methods (nested functions).
  3. Global (G): At the top level of the current module.
  4. Built-in (B): Provided by the built-in namespace.

Namespace Collision

Namespace collisions occur when two namespaces have variables or functions with the same name. This can be avoided by proper namespace management and naming conventions.

Summary

Understanding namespaces is crucial for managing variables and functions in Python effectively. They help in organizing code into manageable scopes and avoiding conflicts between names.
