Python Operator Overloading

Introduction

Operator overloading in Python allows you to define how operators behave for user-defined objects (instances of classes). By overloading operators, you can make your objects work with built-in Python operators like +, -, *, etc.

Basic Syntax

To overload an operator in Python, you define special methods in your class. These methods have predefined names and are called automatically when the corresponding operator is used on instances of your class.

class MyClass:
    def __init__(self, value):
        self.value = value

    def __add__(self, other):
        return self.value + other.value

# Creating instances
obj1 = MyClass(10)
obj2 = MyClass(5)

# Using overloaded operator +
result = obj1 + obj2
print(result)  # Output: 15

Commonly Overloaded Operators

Some commonly overloaded operators in Python include:

  • __add__ for addition (+)
  • __sub__ for subtraction (-)
  • __mul__ for multiplication (*)
  • __truediv__ for true division (/)
  • __floordiv__ for floor division (//)
  • __mod__ for modulus (%)
  • __eq__ for equality (==)
  • __lt__ for less than (<)
  • __gt__ for greater than (>)
  • __str__ for string representation (str())

Example of Overloading

Here's an example demonstrating how to overload the addition operator (+):

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __str__(self):
        return f"({self.x}, {self.y})"

# Creating instances
v1 = Vector(1, 2)
v2 = Vector(3, 4)

# Adding vectors using overloaded operator +
result = v1 + v2
print(result)  # Output: (4, 6)

Special Methods

Special methods (often called magic methods or dunder methods) in Python are surrounded by double underscores (__). These methods provide special functionality like operator overloading, string representation, and more.

Summary

Operator overloading in Python allows you to define how operators should behave for instances of your custom classes. By implementing special methods, you can make your objects work seamlessly with built-in operators, enhancing the flexibility and readability of your code.
