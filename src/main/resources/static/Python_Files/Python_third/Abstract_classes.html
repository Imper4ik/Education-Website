Python Abstract Classes

Introduction

In Python, an abstract class is a class that cannot be instantiated directly. It serves as a blueprint for other classes, providing a common interface and structure that its subclasses must implement.

Creating Abstract Classes

Python abstract classes are created using the abc module and the ABC class as a base. Abstract methods within these classes are declared using the @abstractmethod decorator.

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

    @abstractmethod
    def perimeter(self):
        pass

Subclassing Abstract Classes

Subclasses of abstract classes must implement all abstract methods defined in the abstract class. Failure to do so will result in a TypeError during instantiation.

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius * self.radius

    def perimeter(self):
        return 2 * 3.14 * self.radius

Using Abstract Classes

Abstract classes are used to define a common interface for related classes. They allow for consistency and enforce the implementation of specific methods across different subclasses.

def print_shape_details(shape):
    print(f"Area: {shape.area()}")
    print(f"Perimeter: {shape.perimeter()}")

circle = Circle(5)
print_shape_details(circle)

Benefits of Abstract Classes

Abstract classes promote code reusability and enforce structure in object-oriented design. They define a contract for subclasses, ensuring that they provide necessary functionality while allowing flexibility in implementation.

Summary

Python abstract classes provide a way to define common interfaces and enforce rules for subclasses. They are essential for creating hierarchies of related classes and ensuring consistency in object-oriented programming.
