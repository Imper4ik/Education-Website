Python Access Control

Introduction

Access control in Python refers to mechanisms for restricting access to certain parts of code or data. It helps in maintaining security, data integrity, and encapsulation in programs.

Access Modifiers

Python does not have private or protected keywords like some other programming languages. Instead, it uses conventions to indicate the level of access:

  • Public: Members (variables and methods) are accessible from outside the class. There are no explicit modifiers for public access.
  • Protected: Conventionally, members prefixed with a single underscore (_variable or _method()) are considered protected. They can be accessed within the class itself and its subclasses.
  • Private: Conventionally, members prefixed with a double underscore (__variable or __method()) are considered private. They cannot be accessed directly from outside the class.

Python does not enforce these conventions; they are purely a matter of convention and good practice.

Example of Access Control

Below is an example demonstrating access control using public, protected, and private members:

class MyClass:
    def __init__(self):
        self.public_variable = "Public"   # Public member
        self._protected_variable = "Protected"   # Protected member
        self.__private_variable = "Private"   # Private member

    def public_method(self):
        print("This is a public method.")

    def _protected_method(self):
        print("This is a protected method.")

    def __private_method(self):
        print("This is a private method.")

# Outside the class
obj = MyClass()

# Accessing public members
print(obj.public_variable)
obj.public_method()

# Accessing protected members (conventionally)
print(obj._protected_variable)
obj._protected_method()

# Accessing private members (name mangling)
# This is how Python handles private members access
print(obj._MyClass__private_variable)
obj._MyClass__private_method()

Encapsulation

Access control helps in achieving encapsulation, where implementation details are hidden and only essential features are exposed. This improves code maintainability and reduces complexity.

Summary

Understanding access control in Python is crucial for writing maintainable and secure code. By following conventions for public, protected, and private members, developers can achieve better encapsulation and manage code complexity effectively.
