stdlib.h (cstdlib) in C#

The `stdlib.h` header in C, known as `cstdlib` in C++, provides a set of functions for performing general functions such as memory management, program control, and other utilities. In C#, equivalent functionality is available through various classes and methods in the .NET framework.

Here's a comparison of some common functions from `stdlib.h` and their C# equivalents:

        
#include <stdlib.h>

int main()
{
    // Memory allocation
    int* arr = (int*)malloc(5 * sizeof(int));
    free(arr);

    // Random number generation
    int randomNum = rand();

    // Converting string to number
    int num = atoi("123");

    // Exiting the program
    exit(0);

    return 0;
}

The equivalent code in C# would look like this:

        
using System;

class Program
{
    static void Main()
    {
        // Memory allocation (automatic in C# using arrays)
        int[] arr = new int[5];

        // Random number generation
        Random random = new Random();
        int randomNum = random.Next();

        // Converting string to number
        int num = int.Parse("123");

        // Exiting the program
        Environment.Exit(0);
    }
}

In this example, we see how different features provided by `stdlib.h` in C/C++ are implemented in C#:

  • Memory allocation and deallocation are handled automatically in C# using the garbage collector. We use arrays directly without needing `malloc` and `free`.
  • Random number generation in C# is done using the `Random` class, which provides more functionality than `rand`.
  • String to number conversion is done using `int.Parse` or `Convert.ToInt32`.
  • Exiting the program is done using `Environment.Exit`.
