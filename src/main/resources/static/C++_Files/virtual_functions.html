C++ Virtual Functions

Introduction

Virtual functions in C++ are functions declared in a base class and overridden in derived classes. They enable runtime polymorphism, allowing the correct function to be invoked for an object, regardless of the expression used to call it.

Usage of Virtual Functions

To declare a virtual function in C++, use the virtual keyword in the base class:

class Base {
public:
    virtual void display() {
        cout << "Displaying from Base class" << endl;
    }
};

class Derived : public Base {
public:
    void display() override {
        cout << "Displaying from Derived class" << endl;
    }
};

int main() {
    Base* basePtr = new Derived();
    basePtr->display(); // Calls Derived class's display()
    delete basePtr;
    return 0;
}

In this example, Base has a virtual function display() which is overridden in Derived. The virtual function mechanism ensures that Derived's version of display() is called even though the pointer is of type Base*.

Virtual Destructor

It's important to make the base class destructor virtual if the class is intended to be used polymorphically to ensure proper cleanup of resources:

class Base {
public:
    virtual ~Base() {
        // Virtual destructor
    }
};

Abstract Classes

Virtual functions are commonly used in abstract classes, which are classes that cannot be instantiated on their own but are meant to be inherited:

class AbstractBase {
public:
    virtual void pureVirtualFunction() = 0; // Pure virtual function
    virtual void regularVirtualFunction() {
        // Regular virtual function implementation
    }
};

A class with at least one pure virtual function is called an abstract class and cannot be instantiated directly. Derived classes must provide implementations for all pure virtual functions to be instantiated.

Summary

Virtual functions enable polymorphism in C++, allowing derived classes to provide specific implementations of functions defined in base classes. Understanding virtual functions is crucial for leveraging polymorphism effectively in object-oriented programming.
