C++ Abstract Classes

Introduction

In C++, an abstract class is a class that has at least one pure virtual function. Abstract classes cannot be instantiated directly; they are meant to be inherited by other classes, which must implement all pure virtual functions to become concrete classes.

Declaration of Abstract Classes

To declare an abstract class, you use at least one pure virtual function:

class AbstractBase {
public:
    virtual void pureVirtualFunction() = 0; // Pure virtual function
    virtual void regularVirtualFunction() {
        // Regular virtual function implementation
    }
};

Here, pureVirtualFunction() is a pure virtual function, denoted by = 0, meaning any class inheriting from AbstractBase must provide an implementation for it.

Using Abstract Classes

Abstract classes are used to define interfaces and provide a common base for derived classes:

class Shape {
public:
    virtual void draw() const = 0; // Pure virtual function
    virtual double area() const = 0; // Another pure virtual function
    virtual ~Shape() {} // Virtual destructor
};

class Circle : public Shape {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    void draw() const override {
        cout << "Drawing Circle" << endl;
    }
    double area() const override {
        return 3.14 * radius * radius;
    }
};

int main() {
    // Shape shape; // Error: Cannot instantiate abstract class
    Circle circle(5.0);
    circle.draw();
    cout << "Area: " << circle.area() << endl;
    return 0;
}

In this example, Shape is an abstract base class with pure virtual functions draw() and area(). Circle inherits from Shape and provides concrete implementations for these functions.

Benefits of Abstract Classes

Abstract classes facilitate:

  • Polymorphism: Derived classes can be treated uniformly through pointers or references of the abstract base class.
  • Forcing interfaces: Ensures derived classes implement specific functionality, enforcing design contracts.
  • Code organization: Defines a clear hierarchy of related classes and interfaces.

Summary

Abstract classes in C++ provide a way to define interfaces and enforce the implementation of specific methods in derived classes. They play a crucial role in achieving polymorphism and designing robust object-oriented systems.
