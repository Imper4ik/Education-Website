C++ Destructors

Introduction

Destructors in C++ are special member functions that are executed when an object of a class is destroyed. They are used to release resources that the object may have acquired during its lifetime.

Syntax of Destructors

A destructor has the same name as the class, preceded by a tilde (~), and it cannot take arguments or return a value.

class MyClass {
public:
    ~MyClass() {
        // Destructor code here
    }
};

When are Destructors Called?

Destructors are called automatically in the following scenarios:

  • When an object goes out of scope.
  • When a dynamically allocated object is deleted using the delete operator.
  • When a program ends and global or static objects are destroyed.

Example of a Destructor

Below is an example of a class with a destructor that prints a message when it is called:

class MyClass {
public:
    MyClass() {
        cout << "Constructor called!" << endl;
    }
    ~MyClass() {
        cout << "Destructor called!" << endl;
    }
};

int main() {
    MyClass obj; // Constructor is called
    // Destructor will be called automatically when obj goes out of scope
    return 0;
}

Releasing Resources

Destructors are commonly used to release resources such as memory, file handles, or network connections. Here is an example of a destructor that releases dynamically allocated memory:

class MyClass {
private:
    int* data;
public:
    MyClass(int size) {
        data = new int[size];
        cout << "Memory allocated!" << endl;
    }
    ~MyClass() {
        delete[] data;
        cout << "Memory deallocated!" << endl;
    }
};

int main() {
    MyClass obj(10); // Memory allocated
    // Memory will be deallocated when obj goes out of scope
    return 0;
}

Summary

Destructors are essential for resource management in C++. They ensure that resources are properly released when an object is no longer needed. Understanding how to use destructors effectively is crucial for writing robust and efficient C++ programs.
