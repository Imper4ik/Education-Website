C++ Namespaces

Introduction

Namespaces are used in C++ to organize code into logical groups and to prevent name collisions that can occur especially when your code base includes multiple libraries. Namespaces provide a way to group identifiers (names of types, functions, variables, etc.) under a name.

Defining a Namespace

You define a namespace using the namespace keyword followed by the name of the namespace and a block of code enclosed in curly braces:

namespace myNamespace {
    int myVariable;
    void myFunction() {
        // function body
    }
}

Using a Namespace

To access members of a namespace, you use the scope resolution operator :::

int main() {
    myNamespace::myVariable = 5;
    myNamespace::myFunction();
    return 0;
}

Namespace Aliases

You can create an alias for a namespace to make it easier to work with, especially if the namespace name is long:

namespace mn = myNamespace;

int main() {
    mn::myVariable = 5;
    mn::myFunction();
    return 0;
}

Unnamed (Anonymous) Namespaces

An unnamed or anonymous namespace is a namespace that doesn't have a name. Members of an unnamed namespace have internal linkage, meaning they are only accessible within the file they are declared in:

namespace {
    int myInternalVariable;
    void myInternalFunction() {
        // function body
    }
}

int main() {
    myInternalVariable = 5;
    myInternalFunction();
    return 0;
}

Nested Namespaces

Namespaces can be nested inside other namespaces. You can access members of a nested namespace using the scope resolution operator:

namespace outerNamespace {
    namespace innerNamespace {
        int myVariable;
        void myFunction() {
            // function body
        }
    }
}

int main() {
    outerNamespace::innerNamespace::myVariable = 5;
    outerNamespace::innerNamespace::myFunction();
    return 0;
}

Using Directive

The using directive allows you to use all the members of a namespace without prefixing them with the namespace name:

using namespace myNamespace;

int main() {
    myVariable = 5;
    myFunction();
    return 0;
}

Note that using directives can lead to name collisions and should be used with caution.
