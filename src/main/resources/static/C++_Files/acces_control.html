C++ Access Control

Introduction

Access control in C++ is implemented using access specifiers. These specifiers determine the accessibility of class members (attributes and methods) within and outside the class. The three primary access specifiers are public, private, and protected.

Public Access

Members declared as public are accessible from anywhere in the program where the object is visible.

class MyClass {
public:
    int myNumber;
    void myFunction() {
        cout << "Hello, World!";
    }
};

int main() {
    MyClass obj;
    obj.myNumber = 10; // Accessible
    obj.myFunction();  // Accessible
    return 0;
}

Private Access

Members declared as private are accessible only within the class itself. They cannot be accessed directly from outside the class.

class MyClass {
private:
    int myNumber;
    void myFunction() {
        cout << "Hello, World!";
    }
public:
    void setNumber(int num) {
        myNumber = num;
    }
    int getNumber() {
        return myNumber;
    }
};

int main() {
    MyClass obj;
    // obj.myNumber = 10; // Not accessible
    // obj.myFunction();  // Not accessible
    obj.setNumber(10);   // Accessible through public function
    cout << obj.getNumber(); // Accessible through public function
    return 0;
}

Protected Access

Members declared as protected are accessible within the class itself and by derived classes.

class Base {
protected:
    int myNumber;
public:
    void setNumber(int num) {
        myNumber = num;
    }
};

class Derived : public Base {
public:
    void display() {
        cout << myNumber; // Accessible in derived class
    }
};

int main() {
    Derived obj;
    obj.setNumber(10); // Accessible through public function
    obj.display();     // Accessible in derived class function
    return 0;
}

Summary

Access specifiers are crucial in implementing encapsulation in C++. They help in controlling the visibility and accessibility of class members, thereby protecting the integrity of the data.
