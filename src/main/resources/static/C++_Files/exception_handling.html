Exception Handling in JavaScript

Introduction

Exception handling in JavaScript allows you to gracefully handle runtime errors that may occur during the execution of your code.

Try-Catch Statement

The try-catch statement is used to handle exceptions in JavaScript:

try {
    // Block of code to try
    throw new Error('Custom error message');
} catch (error) {
    // Block of code to handle errors
    console.error('An error occurred:', error.message);
}

Throw Statement

The throw statement allows you to create a custom error:

function divide(x, y) {
    if (y === 0) {
        throw new Error('Division by zero');
    }
    return x / y;
}

try {
    console.log(divide(10, 2)); // Output: 5
    console.log(divide(10, 0)); // This will throw an error
} catch (error) {
    console.error('An error occurred:', error.message);
}

Finally Block

The finally block is executed regardless of whether an exception is thrown or caught:

try {
    // Block of code to try
    throw new Error('Custom error message');
} catch (error) {
    // Block of code to handle errors
    console.error('An error occurred:', error.message);
} finally {
    // Block of code to execute regardless of errors
    console.log('Cleanup code here');
}

Summary

Exception handling in JavaScript using try-catch statements allows you to manage errors effectively, improving the robustness and reliability of your applications.
